# RAG ПТЭ/ИДП — Система поиска нарушений

RAG-система для автоматического поиска нарушений Правил технической эксплуатации (ПТЭ) и Инструкции по движению поездов и маневровой работе (ИДП) железных дорог РФ.

## Описание

Система принимает замечание о нарушении и находит соответствующий пункт нормативного документа. Если точный пункт не определён — предлагает кандидатов через LLM.

## Архитектура

```
Замечание → Keyword Search → Fuzzy Search → LLM Candidates → User Selection → Formal Remark
```

| Этап | Метод | Описание |
|------|-------|----------|
| 1 | KEYWORD | Точное совпадение по ключевым словам (100% точность) |
| 2 | FUZZY | Fuzzy matching по 176 правилам |
| 3 | NEED_SELECTION | LLM предлагает 2-5 кандидатов |
| 4 | NOT_FOUND | Пункт не определён |

## Что сделано (2026-04-21)

### Реализовано
1. **rules.py** — comprehensive база правил для ПТЭ и ИДП
   - 176 правил covering все разделы ПТЭ (1-9) и приложения ИДП (6,7,9-14,16)
   - Keyword-to-punkt mapping для точного поиска
   - Rule dataclass с metadata (section_desc, rule_text)

2. **rag_system.py** — многоуровневая система поиска
   - Keyword matching (точное)
   - Fuzzy search (по всем правилам)
   - LLM fallback для неопределённых случаев
   - Генерация формальных замечаний в деловом стиле

3. **streamlit_app.py** — web интерфейс
   - Ввод замечания
   - Автоматический поиск
   - UI для выбора кандидатов (radio buttons)
   - Формирование официального замечания
   - Отображение полного текста пункта

4. **Приложение 7 ИДП** — добавлено 9 правил
   - Перерывы систем интервального регулирования
   - Восстановительные поезда
   - Ключевое правило: п.6 — обязанность убедиться в отсутствии поездов

### Тестирование
| Замечание | Результат | Метод |
|-----------|-----------|-------|
| "машинист ехал без свидетельства" | Раздел 2 п.9 ПТЭ | KEYWORD ✅ |
| "видимость светофора менее 1000м" | Раздел 6 п.74 ПТЭ | KEYWORD ✅ |
| "разграничение временем не установлено" | Приложение 6 п.1 ИДП | KEYWORD ✅ |
| "не убедился в отсутствии поездов" | Приложение 7 п.6 ИДП | KEYWORD ✅ |
| "машинист не проверил тормоза" | 2 кандидата | LLM |

### Известные ограничения
- LLM fallback даёт 40-60% точность (галлюцинации, игнорирует формат)
- Ручное создание правил: 176 правил = ~20 часов работы
- При масштабировании на 1000+ документов потребуется полная переработка
